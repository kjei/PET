Documentation generation

The documentation is generated using pdoc3, which grabs the docstrings of modules/classes/functions, and renders them into pretty html. The docstrings should be written using markdown syntax, with some extras (cross-referencing, math, etc)

Run pdoc locally

Unlike the typical case with sphinx, we do not use readthedocs for generation and hosting, but rather GitHub Actions and Pages. To preview you changes to the docs, do the following (manually including all of the folders/packages will become unnecessary once they've all been put into a root folder (which they should be)):

pdoc --force --html --template-dir docs/templates -o ./docs/ \
    docs/dev_guide.py ip.py fwd_sim input_output geostat optimization

Open docs/index.html to preview the result.

There is also a nice --http flag to get live reloading as you make changes. Use it for example as follows:

pdoc --template-dir docs/templates --http : ip.py

Tests

Run the tests by the command

pytest

It will discover all appropriately named tests in the source (see the tests dir).

Use (for example) pytest --doctest-modules docs/dev_guide.py to also run any example code within docstrings.

The CI also simply runs pytest.
